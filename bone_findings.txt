The T-pose is because LeftArm z=0.5646 (≈ 1.18 radians ≈ 67.5°) and RightArm z=-0.5646.
To bring arms down, we need z closer to 0 for both arms.

In Three.js Euler rotation (which is what the code uses):
- LeftArm z=1.2 means rotating arm DOWN from T-pose (correct direction)
- RightArm z=-1.2 means rotating arm DOWN from T-pose (correct direction)

But the GLB stores quaternions. The current code sets z=1.2 and z=-1.2 which should work.
The issue might be that Object.assign on rotation doesn't trigger the matrix update.

FIX: Use rotation.set(x, y, z) instead of Object.assign, and call updateMatrixWorld.
